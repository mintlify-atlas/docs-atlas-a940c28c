Faker is installed as a development dependency. This guide walks you through installation, basic usage, locale selection, seeding for reproducible results, and building factory functions for complex objects.

<Steps>
  <Step title="Install the package">
    Install `@faker-js/faker` using your preferred package manager.

    <CodeGroup>
      ```bash npm
      npm install --save-dev @faker-js/faker
      ```
      ```bash pnpm
      pnpm add --save-dev @faker-js/faker
      ```
      ```bash yarn
      yarn add --dev @faker-js/faker
      ```
    </CodeGroup>
  </Step>

  <Step title="Generate your first fake data">
    Import `faker` and call any of its module methods to generate data.

    ```typescript
    import { faker } from '@faker-js/faker';

    const user = {
      userId: faker.string.uuid(),
      name: faker.person.fullName(),
      email: faker.internet.email(),
      avatar: faker.image.avatar(),
      birthdate: faker.date.birthdate(),
      registeredAt: faker.date.past(),
    };

    console.log(user);
    ```

    Each call returns a freshly generated random value. Run the script multiple times and you will get different results each time.
  </Step>

  <Step title="Use a locale">
    The default `faker` instance generates English data. To generate locale-specific values, import a pre-configured locale instance.

    ```typescript
    import { fakerDE as faker } from '@faker-js/faker';

    faker.person.firstName(); // 'Lena'
    faker.location.city();    // 'München'
    ```

    Over 70 locale instances are available, named after their [BCP 47](https://www.rfc-editor.org/rfc/bcp/bcp47.txt) language tag — for example `fakerFR`, `fakerJA`, `fakerPT_BR`. See the [Localization guide](/guides/localization) for the full list.
  </Step>

  <Step title="Seed for reproducible results">
    By default, Faker produces a different value on every run. To get the same value every time — useful for snapshot tests — call `faker.seed()` with a fixed integer before generating data.

    ```typescript
    import { faker } from '@faker-js/faker';

    faker.seed(123);

    const firstRun = faker.number.int(); // always the same value

    // Resetting the seed restarts the sequence from the beginning.
    faker.seed(123);

    const secondRun = faker.number.int();

    console.log(firstRun === secondRun); // true
    ```

    <Note>
      Seeded output may change between major Faker versions if the underlying locale data changes. Pin your Faker version in `package.json` if exact reproducibility across upgrades is required.
    </Note>
  </Step>
</Steps>

## Creating complex objects

Faker generates values for individual primitives. For structured objects, write a factory function that calls multiple Faker methods and returns a typed record.

The example below shows how to build a `createRandomUser` factory. It generates consistent related values by deriving `firstName`, `lastName`, and `email` in the correct order so the email address reflects the person's name.

```typescript
import { faker } from '@faker-js/faker';
import type { SexType } from '@faker-js/faker';

type SubscriptionTier = 'free' | 'basic' | 'business';

interface User {
  _id: string;
  avatar: string;
  birthday: Date;
  email: string;
  firstName: string;
  lastName: string;
  sex: SexType;
  subscriptionTier: SubscriptionTier;
}

function createRandomUser(): User {
  const sex = faker.person.sexType();
  const firstName = faker.person.firstName(sex);
  const lastName = faker.person.lastName();
  const email = faker.internet.email({ firstName, lastName });

  return {
    _id: faker.string.uuid(),
    avatar: faker.image.avatar(),
    birthday: faker.date.birthdate(),
    email,
    firstName,
    lastName,
    sex,
    subscriptionTier: faker.helpers.arrayElement(['free', 'basic', 'business']),
  };
}

const user = createRandomUser();
```

To generate a list of users, use `faker.helpers.multiple`:

```typescript
const users = faker.helpers.multiple(createRandomUser, { count: 10 });
```

<Tip>
  Add an optional `overwrites` parameter to your factory function to allow individual properties to be overridden in tests. This lets you control just the values relevant to a given test case without having to construct the full object by hand.
</Tip>
