`Faker` is the main class for generating fake data. The default `faker` export is a pre-built English instance ready to use with no setup. Use `new Faker({ locale })` only when you need a custom locale or a specific fallback chain.

## Quick usage

<CodeGroup>

```typescript ESM
import { faker } from '@faker-js/faker';

faker.person.fullName(); // 'John Doe'
faker.internet.email();  // 'john.doe@example.com'
```

```javascript CJS
const { faker } = require('@faker-js/faker');

faker.person.fullName(); // 'John Doe'
faker.internet.email();  // 'john.doe@example.com'
```

</CodeGroup>

<Note>
For tree-shaking and smaller bundles, import from a locale-specific path:

```typescript
import { faker } from '@faker-js/faker/locale/en';
```

This loads only the English locale data and cuts out all other locale definitions.
</Note>

## Constructor

Use the constructor when you need a custom locale, a fallback chain, or a specific randomizer. For most use cases a pre-built instance such as `faker`, `fakerDE`, or `fakerES_MX` is sufficient.

```typescript
import { Faker, de, de_AT, en, base } from '@faker-js/faker';

const customFaker = new Faker({ locale: [de_AT, de, en, base] });

customFaker.person.firstName(); // 'Gerhard'
customFaker.location.city();    // 'Wien'
```

### Options

<ParamField body="locale" type="LocaleDefinition | LocaleDefinition[]" required>
  The locale data to use. Pass an array to define a fallback chain ŌĆö Faker uses the first locale that has data for each requested property. Include `base` at the end to cover locale-independent entries.

  ```typescript
  // Single locale
  new Faker({ locale: de });

  // Fallback chain: de_AT ŌåÆ de ŌåÆ en ŌåÆ base
  new Faker({ locale: [de_AT, de, en, base] });
  ```
</ParamField>

<ParamField body="randomizer" type="Randomizer">
  A custom randomizer to use instead of Faker's default Mersenne Twister (53-bit). Use this when you want to share a random generator with another library or tool.

  ```typescript
  import { generateMersenne32Randomizer } from '@faker-js/faker';

  const randomizer = generateMersenne32Randomizer();
  const customFaker = new Faker({ locale: en, randomizer });
  ```
</ParamField>

<ParamField body="seed" type="number | number[]">
  An initial seed for reproducible output. Equivalent to calling `faker.seed(n)` immediately after construction.

  ```typescript
  const seededFaker = new Faker({ locale: en, seed: 42 });
  seededFaker.number.int(100); // always the same value for seed 42
  ```
</ParamField>

## Methods

### seed()

Sets the seed for the random number generator to produce reproducible output. Generated values depend on both the seed and the number of calls made since it was set.

```typescript
faker.seed(42);
faker.number.int(10); // 4
faker.number.int(10); // 10

faker.seed(42);
faker.number.int(10); // 4  ŌåÉ same as before
faker.number.int(10); // 10 ŌåÉ same as before
```

Pass an array for additional entropy:

```typescript
faker.seed([42, 13, 17]);
faker.number.int(10); // 3
```

Call `seed()` with no argument to generate a random seed and get back its value ŌĆö useful for logging and reproducing failing tests:

```typescript
const seed = faker.seed();
console.log('seed:', seed); // seed: 3487219384
```

<ParamField body="seed" type="number | number[]">
  The seed value. Pass a single number or an array of numbers. Omit to generate a random seed. Returns the seed that was set.
</ParamField>

### setDefaultRefDate()

Sets the reference date used as "now" when generating relative dates such as `faker.date.past()` or `faker.date.future()`. Defaults to `() => new Date()` evaluated at call time.

Use this together with `seed()` to make date generation fully reproducible:

```typescript
faker.seed(1234);
faker.setDefaultRefDate(new Date('2020-01-01'));

faker.date.past(); // always '2019-07-03T08:27:58.118Z' for this seed + refDate
```

<ParamField body="dateOrSource" type="string | Date | number | (() => Date)" default="() => new Date()">
  A static value (date string, `Date` object, or Unix timestamp in milliseconds) or a function that returns a fresh `Date` on each call. Omit or pass `undefined` to restore the default behavior.
</ParamField>

### getMetadata()

Returns an object describing the locale loaded in this `Faker` instance.

```typescript
import { faker, fakerES_MX } from '@faker-js/faker';

faker.getMetadata();
// { title: 'English', code: 'en', language: 'en', endonym: 'English', dir: 'ltr', script: 'Latn' }

fakerES_MX.getMetadata();
// { title: 'Spanish (Mexico)', code: 'es_MX', language: 'es', endonym: 'Espa├▒ol (M├®xico)', dir: 'ltr', script: 'Latn', country: 'MX' }
```

<Note>
`getMetadata()` is only available on `Faker`, not on `SimpleFaker`. If the loaded locale does not define metadata, an empty object is returned.
</Note>
